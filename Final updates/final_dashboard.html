cat Command


The cat command is the simplest command to use when you want to see the contents of a particular file. The only issue is that it simply unloads the entire file to your terminal. If you want to navigate around a huge file, should use less command alternatively.

NORMAL OPERATION

Syntax: cat [option] [file]

[option] represents various command-line options. [file] represents the name of the file to be processed.

COMMON OPTIONS

1. -n: introduces line numbers, making it convenient to identify and reference specific lines within the file.
2. -s: comes in handy when dealing with files containing repeated empty lines. It suppresses these repetitions, providing a cleaner output.
3. -E: used to highlight the end of each line.
4. -v: creates non-printing characters visible (except for tabs and line breaks).
5. -T: shows tabs as ^I.

awk Command


The awk command is a powerful text processing tool in Unix/Linux used for pattern scanning and processing. It allows you to manipulate data and generate reports by processing input line by line, which can be used for tasks like searching, extracting, and modifying text from files or streams.

Awk is like a Swiss army knife for text, making it easy to dissect, rearrange, and analyze data.

NORMAL OPERATION

Syntax: awk [options] '[pattern] {action}' [file]

[options]: Optional flags to modify awk behavior. [pattern]: The condition or pattern to match. [action]: The operation to perform on lines that match the pattern. [file]: The file or files to process. If no file is specified, `awk` reads from the standard input.

COMMON OPTIONS

1. -F: Specify a custom field separator (default is whitespace).
2. -v: Set variables to be used within awk.
3. -f: Read awk commands from a file.
4. -W: Enable or disable specific features.

cd Command


The cd command allows you to change directories in Linux, making it easier to navigate through the file system and manage your files efficiently.

I'm really good at cd-ing to the right place... I guess you could say I always navigate well.

MOVE TO THE SUB-DIRECTORY

Syntax: cd {subdirectory_name}

This will put the user in a subdirectory. For example, if you are in the directory /name, execution of 'cd my_name' will take you to /name/my_name.

MOVE TO THE PARENT-DIRECTORY

Syntax: cd ..

This will move the user up one directory. For example, if you are in the directory /name/my_name, execution of 'cd ..' will take you to /name.
